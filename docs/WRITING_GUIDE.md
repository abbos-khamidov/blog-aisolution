# Как писать для blog.aisolution.uz

## Что писать

Мнение AI Solution по теме. Не пересказ новости — позиция.

Структура: тезис → аргумент → пример из практики → вывод. Личный тезис должен
быть виден уже в первых двух абзацах — если его нет, это не блог, а рерайт.

Объём: 700–1500 слов.

## Что нельзя

- Копировать текст из других источников (цитата — до 15 слов, с атрибуцией).
- Политика, госрегулирование, оценка действий властей.
- Цифры клиентских проектов без согласования с founder.
- Обещания результата («ИИ увеличит продажи на 300%»).
- Статья без личного тезиса — это не блог, это рерайт.

## Workflow

1. `npm run new-post` — интерактивный скаффолд спросит язык, заголовок,
   описание, ключевые слова, автора (из `lib/authors.ts`) и кластер.
   Он сам транслитерирует slug и сразу проверяет фронтматтер через
   `lib/schema.ts` — если что-то не так, скрипт откажется создать файл.
2. Пишешь текст в сгенерированном `content/<lang>/<slug>.mdx`, заменяя
   плейсхолдеры тезис/аргумент/пример/вывод реальным текстом.
3. Когда готово — `draft: false` в фронтматтере, затем `npm run build`.
   Билд должен пройти локально без ошибок.
4. Если пишешь и перевод (ru↔uz), свяжи статьи через `translationOf` —
   значение равно `slug` другой языковой версии.
5. PR в `main`, ревью founder, мерж.
6. `./deploy.sh` на сервере (это делает founder / DevOps, не автор статьи).

## Твой автор в реестре

Если тебя ещё нет в `lib/authors.ts` — без записи там статья не пройдёт
валидацию (`author` должен быть реальным ключом из реестра). Попроси
founder добавить тебя: `key`, `name.ru`/`name.uz`, `role`, `bio`, `image`,
`isFounder: false`.

## Чеклист перед PR

- [ ] Заголовок 20–70 символов
- [ ] Описание (meta description) 70–160 символов
- [ ] `topicCluster` выбран осмысленно, не первый попавшийся
- [ ] 1–8 keywords
- [ ] `cover` указывает на реальный файл в `/public`
- [ ] Личный тезис есть в первых двух абзацах
- [ ] Нет цифр клиентских проектов без согласования
- [ ] Нет обещаний конкретного результата
- [ ] `draft: false`
- [ ] `npm run build` проходит локально
